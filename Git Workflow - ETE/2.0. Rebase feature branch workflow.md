# Branches - workflows
- Do your updates in a branch, not main

# Branches - No Rebase
1. Pre-branch: `git switch main`, then `git pull` + enter
2. Create and Switch: `git switch -c <feature/feature-name>` + enter
3. Develop:
    - Add Source Code
    - Stage: `git add .` or `git add <file>`
    - Commit: `git commit -m "message"`
    - Push: `git push -u origin <feature/feature-name>`
4. GitHubUI: Open PR → Review → Approve → Merge
5. Sync Local:
    - Switch to main: `git switch main`
    - Review First: Fetch: `git fetch origin main` → Review:`git diff main origin/main` → Merge`git merge origin/main` OR
    - Quick Sync: `git pull origin main`
6. Post-merge Cleanup:
    - Delete remote `git push origin --delete <feature/feature-name>`
    - Delete local `git branch -d <feature/feature-name>` (or `-D` if needed)
    - Prune stale references `git remote prune origin`

# Branches - Rebase
1. Pre-branch: `git switch main`, then `git pull` + enter
2. Create and Switch: `git switch -c <feature/feature-name>` + enter
3. Develop:
    - Add Source Code
    - Stage: `git add .` or `git add <file>`
    - Commit: `git commit -m "message"`
    - Push: `git push -u origin <feature/feature-name>`
4. GitHubUI: Update PR (if open) → Review → Approve → Merge
5. Sync Local:
    - Switch to main: `git switch main`
    - Review First: Fetch: `git fetch origin main` → Review:`git diff main origin/main` → Merge`git merge origin/main` OR
    - Quick Sync: `git pull origin main`
6. Rebase:
    - Switch: `git switch <feature/feature-name>`
    - Fetch: `git fetch origin`
    - Rebase: `git rebase origin/main`
    - Resolve Conflicts if any:
        - Mark conflicts as resolved: `git add <file>`
        - Continue Rebase:`git rebase --continue`
7. Develop:
    - Add Source Code
    - Stage: `git add .` or `git add <file>`
    - Commit: `git commit -m "message"`
    - Push after Rebase:
        - If branch was already pushed → `git push --force-with-lease origin <feature/feature-name>`
        - If first push after rebase → `git push origin <feature/feature-name>`
8. GitHubUI: Open PR → Review → Approve → Merge
9. Sync Local:
    - Switch to main: `git switch main`
    - Review First: `git fetch origin main`, review, `git merge` OR
    - Quick Sync: `git pull origin main`
10. Post-merge Cleanup:
    - Delete merged remote `git push origin --delete <feature/feature-name>`
    - Delete merged local `git branch -d <feature/feature-name>` (or `-D` if needed)
    - Prune stale references `git remote prune origin`
